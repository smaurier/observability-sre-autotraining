# Lab 12 - Gestion d incidents et postmortems

## Objectifs

- Modeliser un incident avec ses proprietes (severite, statut, timeline, roles)
- Implementer une machine a etats pour le cycle de vie d un incident
- Comprendre les roles cles (IC, Comms Lead, Ops Lead)
- Maitriser la technique des 5 Whys
- Construire un diagramme d Ishikawa
- Definir des action items SMART
- Generer un document postmortem complet

## Exercices

1. **Modeliser un incident** - type Incident avec severity, status, timeline, roles
2. **Machine a etats** - transitions detected -> triaged -> mitigating -> resolved
3. **Assigner les roles** - IC, Comms Lead, Ops Lead
4. **Timeline de communication** - mises a jour horodatees
5. **Les 5 Whys** - chaine de pourquoi jusqu a la cause racine
6. **Diagramme d Ishikawa** - causes par categories
7. **Action Items SMART** - validation des action items
8. **Postmortem complet** - generer un document complet
